# STEP B2 — Deterministic Decision Core

**From Concept → Rules → Scores → Explainability**

> Ziel:
> Eine **auditierbare Entscheidungsmaschine**, die
>
> * alle Phasen strikt einhält
> * SAFE / EMOTIONAL / BOLD **ableitet**, nicht auswählbar macht
> * vollständig ohne Produktdaten funktioniert
> * Explainability **regelbasiert** erzeugt

---

## 0. Architektur-Leitplanken (unverhandelbar)

* **Keine probabilistischen Entscheidungen**
* **Keine Generierung**
* **Keine UI-Abhängigkeit**
* **Keine Memory-Pflicht** (Memory wirkt nur modulierend)
* **Feste Phasenreihenfolge** (kein Skippen)

Decision Core = **Pure Function + erklärbare Seiteneffekte**

---

## 1. Normalized Input Object (Intent Lock)

```ts
DecisionContext {
```

➡️ **Nach diesem Punkt ist alles gelockt.**
Keine Interpretation mehr, nur Regeln.

---

## 2. Phase Engine (fixe Reihenfolge)

### Phase 1 — Uncertainty Qualification

**Output:** `decision_risk_level`

**Regeln (Beispiele):**

* hohe Nähe + hohe Bedeutung → Risiko ↑
* wenig Zeit + hohe Erwartung → Risiko ↑
* niedrige Bedeutung → Risiko ↓

```ts
decision_risk_level = LOW | MEDIUM | HIGH
```

➡️ LOW kann Phasen verkürzen, **nie überspringen**.

---

### Phase 2 — Social Expectation Mapping

**Output:** `expectation_frame`

Indikatoren:

* occasion_formality
* cultural_norms
* relationship_expectation

```ts
expectation_frame = CONSERVATIVE | BALANCED | OPEN
```

➡️ Definiert **Abweichungstoleranz**, nicht Kreativität.

---

### Phase 3 — Relationship Risk Profiling

**Output:** `risk_profile`

```ts
risk_profile {
  disappointment_risk
  misunderstanding_risk
  overstep_risk
  insignificance_risk
}
```

Gewichtung aus:

* relationship_type
* closeness
* historical_success (falls vorhanden)

➡️ **dominant_risk_type** wird festgelegt.

---

### Phase 4 — Personality & Fit Matrix

*(Kapitel 25, strikt umgesetzt)*

**Kein Typing. Kein Labeling.**

```ts
fit_vector {
  novelty_tolerance
  emotional_openness
  symbolic_affinity
  practicality_bias
}
```

Mapping:

* personality_traits[]
* surprise_tolerance
* values

➡️ Fit wirkt **einschränkend**, nicht kreativ.

---

### Phase 5 — Pattern Eligibility Filtering

**Hard Filters (Elimination):**

* no_gos
* cultural exclusion
* occasion incompatibility
* non_repetition_rules (STEP 0.8)

**Soft Filters (Penalty):**

* fit deviation
* risk amplification
* personality mismatch

```ts
eligible_patterns[]
```

➡️ Viele Patterns verschwinden **unsichtbar**.

---

### Phase 6 — Risk-Weighted Scoring

*(Kapitel 26)*

Für jedes verbleibende Pattern:

```ts
pattern_score =
  base_weight
+ fit_bonus
- risk_penalty
+ historical_success_boost
```

**Wichtig:**

* kein „Best“
* nur **Verantwortbarkeit**

---

## 3. Confidence Derivation (SAFE / EMOTIONAL / BOLD)

Jetzt erst wird gerahmt:

```ts
SAFE:
  minimizes dominant_risk_type

EMOTIONAL:
  balances risk + meaning

BOLD:
  allows risk consciously
```

**Regeln:**

* max. 3 Optionen
* manchmal nur 2 (wenn Frame es erzwingt)
* nie mehr

➡️ Optionen sind **Entscheidungshaltungen**, keine Produkte.

---

## 4. Explainability Binding (regelbasiert)

Für jede Option wird erzeugt:

```ts
Explanation {
  why_this_works        // aus risk + fit + occasion
  emotional_signal      // Bedeutungsebene
  things_to_consider[]  // explizit erlaubte Risiken
  underlying_pattern
}
```

➡️ **Keine Textidee. Nur Regelableitung.**
