## 1️⃣ Supabase-Queries (UI-Read)

**Eine Tabelle, zwei Rollen** (`q_value_constraints`):

### A) Values

```sql
SELECT code, label
FROM q_value_constraints
WHERE type = 'value'
ORDER BY label;
```

### B) No-Gos

```sql
SELECT code, label
FROM q_value_constraints
WHERE type = 'no_go'
ORDER BY label;
```

> Falls `type` anders heißt (z. B. `constraint_type`): bitte anpassen.
> **Keine weiteren Filter** (kein LIMIT, keine Whitelist).

## 2️⃣ Replit – Fetch Pattern (Screen 3)

```ts
// lib/loadScreen3Options.ts
import { supabase } from './supabase';

export async function loadScreen3Options() {
  const [values, noGos] = await Promise.all([
    supabase
      .from('q_value_constraints')
      .select('code,label')
      .eq('type', 'value')
      .order('label'),

    supabase
      .from('q_value_constraints')
      .select('code,label')
      .eq('type', 'no_go')
      .order('label')
  ]);

  if (values.error || noGos.error) {
    throw new Error('Failed to load Screen 3 options');
  }

  return {
    values: values.data,
    noGos: noGos.data
  };
}
```

## 5️⃣ Persistenz (`recipients` ergänzen)

> Screen 3 **ergänzt** den bestehenden Recipient.

```ts
// services/saveRecipientConstraints.ts
import { supabase } from './supabase';

export async function updateRecipientConstraints(
  recipientId: string,
  data: { values: string[]; no_gos: string[] }
) {
  const { error } = await supabase
    .from('recipients')
    .update({
      values: data.values,
      no_gos: data.no_gos
    })
    .eq('recipient_id', recipientId);

  if (error) throw error;
}
```
