## ✅ Schritt A – Exakte Replit-Anweisung (1:1 umsetzen)

### 1️⃣ Service anlegen (oder erweitern)

**Datei:**
`services/supabase/decisionOptionsService.ts`

```ts
import { supabase } from './client';

export async function loadAllowedDecisionOptions(lifeStageCode: string) {
  const { data, error } = await supabase
    .from('v_allowed_decision_options')
    .select('decision_option')
    .eq('life_stage_code', lifeStageCode)
    .eq('is_allowed', true)
    .order('decision_option');

  if (error) throw error;

  return data.map(d => d.decision_option);
}
```

🔒 **Wichtig**

* keine Fallback-Daten
* kein Mapping
* kein Filter im Code

---

### 2️⃣ Screen-Logik (Decision Screen)

**State**

```ts
const [allowedOptions, setAllowedOptions] = useState<string[]>([]);
const [selectedOption, setSelectedOption] = useState<string | null>(null);
```

---

### 3️⃣ Laden beim Screen-Entry

```ts
useEffect(() => {
  if (!lifeStageCode) return;

  loadAllowedDecisionOptions(lifeStageCode)
    .then(setAllowedOptions)
    .catch(() => setAllowedOptions([]));
}, [lifeStageCode]);
```

🔒 **Regel**

* Wenn `allowedOptions.length === 0` → Screen blockiert
* Das ist **ein Datenfehler**, kein UI-Problem

---

### 4️⃣ Rendering (ohne Logik!)

```tsx
{allowedOptions.map(option => (
  <SelectableCard
    key={option}
    label={option.toUpperCase()}
    selected={selectedOption === option}
    onPress={() => setSelectedOption(option)}
  />
))}
```

❌ KEIN:

* `if (lifeStage === 'baby')`
* `filter()`
* `switch`
* `map-to-label`

---

### 5️⃣ Continue-Button

```tsx
<Button

    await supabase

      .upsert({
        relationship_id,

    navigateNext();

```
