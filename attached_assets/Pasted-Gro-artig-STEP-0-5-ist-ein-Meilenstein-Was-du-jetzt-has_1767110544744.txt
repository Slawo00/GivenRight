## 📣 EXAKTER PROMPT FÜR REPLIT – STEP 0.6

> **Diesen Prompt kannst du 1:1 an Replit geben**

```
STEP 0.6 – Object Pattern Selection (Expo / Cross-Platform)

Goal:
Implement the Object Pattern Selection step that comes AFTER the user
has selected a decision direction (safe / emotional / bold).

IMPORTANT:
This must work for iOS, Android, and Web using Expo.
No native code. No platform-specific APIs.

Functional Requirements:

1. New Screen: ObjectPatternSelectionScreen
   Route: /decision/patterns

2. Input:
   - selectedDirection (safe | emotional | bold)
   - loaded from decision state (useDecisionState)

3. Data Source:
   - Fetch object patterns from Supabase table: object_patterns
   - Filter by direction = selectedDirection
   - Fallback to local mock patterns if Supabase unavailable

4. UI Rules:
   - Show ONLY patterns belonging to the selected direction
   - No products
   - No prices
   - No shops
   - No brand names

5. Each Pattern Card must show:
   - pattern_title
   - short_description (why this works)
   - emotional_intent (1 short line)
   - icon (emoji or simple vector)
   - CTA button: “Explore this type”

6. User Interaction:
   - When user taps a pattern:
     - Save selectedPattern in decision state
     - Navigate to PatternExplanationScreen (STEP 0.6.1)

7. Technical Rules:
   - Fully Expo compatible (iOS / Android / Web)
   - Typed with existing TypeScript types
   - Works with Expo Router
   - No hardcoded strings (use Supabase ui_texts where available)

8. UX Tone:
   - Calm
   - Reassuring
   - Educational
   - Zero sales language

Deliverables:
- Screen component
- Supabase query logic
- State update logic
- Navigation integration
```
