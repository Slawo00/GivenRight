### 🔧 REPLIT PROMPT — STEP 0.4.A

```
We are building the GivenRight app.

Goal:
Set up the INITIAL Supabase configuration database.
This database stores ONLY configuration and content.
NO user data, NO learning, NO writes from the app yet.

This must be compatible with:
- Expo (React Native)
- iOS / Android / Web
- Supabase Postgres
- Multi-language support (starting with English only)

----------------------------------
CREATE THE FOLLOWING TABLES
----------------------------------

1) ui_texts
Purpose: All UI texts, labels, explanations, multi-language.

Fields:
- id (uuid, primary key, default uuid_generate_v4())
- key (text, not null)              -- e.g. "home.title"
- language (text, not null)         -- "en" (future: "de", "fr")
- value (text, not null)
- context (text)                    -- screen or component name
- created_at (timestamp, default now())

Indexes:
- unique(key, language)

----------------------------------

2) decision_parameters
Purpose: All numeric parameters for decision logic.

Fields:
- id (uuid, primary key)
- key (text, unique, not null)      -- e.g. "weight.occasion.birthday"
- value (numeric, not null)
- description (text)

Example keys:
- weight.relationship
- weight.occasion.birthday
- weight.occasion.valentines
- threshold.bold_allowed
- penalty.surprise_disliked

----------------------------------

3) decision_explanations
Purpose: Text explanations per direction (SAFE / EMOTIONAL / BOLD).

Fields:
- id (uuid, primary key)
- direction (text, check in ('safe','emotional','bold'))
- language (text)
- title (text)
- body (text)
- risk_note (text)

----------------------------------

4) object_patterns
Purpose: Pattern-level explanations BEFORE products.

Fields:
- id (uuid, primary key)
- pattern_key (text)               -- e.g. "shared_experience"
- direction (text)
- language (text)
- title (text)
- description (text)

----------------------------------

RULES:
- Enable Row Level Security (RLS)
- Create READ-ONLY policies for anon + authenticated users
- No insert/update/delete from client allowed
- This database is configuration-only in Phase 0

DELIVERABLE:
- SQL schema
- RLS policies
- Short confirmation that this works with Expo iOS/Android/Web
```
